## FASE 1 — Plan de Contenido: Serie Manual

### Premisa narrativa
> *"Empecé con $200. Sin apalancamiento exagerado. Sin magia. Solo el sistema."*

No es un challenge de influencer — es una demostración pública de metodología.
Cada video es una clase magistral envuelta en un trade real.

---

### Canal WhatsApp (150 traders) — Calentamiento

**Objetivo:** Dar valor primero → generar reciprocidad → abrir conversaciones de venta de CCI de forma natural.

**Formato de cada entrega:**
```
1. Contexto del mercado (2-3 min)  →  "¿Por qué estoy mirando esto hoy?"
2. Setup del trade (5-7 min)       →  Entrada, stop, target, razón
3. Ejecución en vivo (3-5 min)     →  Pantalla completa, sin cortes
4. Desglose post-trade (5-7 min)   →  Qué salió bien, qué no, por qué
```

**Frecuencia:** 3-4 veces por semana (no todos los días — escasez percibida).

**Mensaje al grupo antes de cada video:**
> *"Hoy operé esto. Grabé todo. Te explico exactamente el razonamiento detrás."*

---

### Instagram — Arquitectura de contenido

Conversión de 1 video largo → múltiples piezas:

```
Video largo (15-20 min)
        │
        ├── Reel 1: Hook del setup  (30-45 seg)    → "Por qué entré aquí"
        ├── Reel 2: Ejecución pura  (20-30 seg)    → Pantalla, entrada, resultado
        ├── Reel 3: El error / lección (45-60 seg) → Si algo salió mal, mejor
        ├── Reel 4: Educativo (60 seg)             → Concepto técnico del trade
        └── Story: Balance actualizado del día      → "$200 → $XXX. Día N."
```

**Meta:** 3-4 reels/día en días con trade. Días sin trade = reels educativos del backlog.

---

### Calendario — Semana 1

| Día | Acción |
|-----|--------|
| Día 1 | Abrir cuenta $200. Primer video al grupo: "Inicio la serie. Aquí está por qué." |
| Día 2 | Primer trade grabado → WhatsApp + 3 reels |
| Día 3 | Reel educativo del concepto usado en el trade del día 2 |
| Día 4 | Segundo trade o "Update de la cuenta: así va el día 4" |
| Día 5 | Reel de resumen semanal: Balance, lecciones, próximos pasos |

---

### Señal de conversión a CCI

Después de 2-3 semanas de valor constante:

> *"Esto que ven es la estrategia base. Castle Crypto Income es esto mismo, pero estructurado en pools de liquidez con rendimiento pasivo. Si quieres entender cómo funciona, tengo conversaciones abiertas esta semana."*

**Regla:** No vender en cada video. Vender una vez que la audiencia ya te debe atención.

---

### Milestones

#### M0 — Fundación (Semana 1-2)
El bot puede leer mercado y generar señales, sin ejecutar dinero real.

- [ ] Price feed conectado (DEX data: Uniswap, Revert)
- [ ] Indicators Engine (los mismos indicadores de la estrategia manual)
- [ ] Strategy Engine: reglas de entrada/salida codificadas
- [ ] Modo paper trading — simula sin dinero real
- [ ] Logger de decisiones: "Habría entrado aquí, por esta razón"

**Prerrequisito para avanzar a M1:** 2 semanas en papel con winrate >= winrate histórico real.

---

#### M1 — Ejecución controlada (Semana 3-4)
El bot ejecuta trades reales con guardarraíles duros.

- [ ] Exchange Connector (API del exchange)
- [ ] Position Manager — límites de exposición máxima
- [ ] Risk Manager:
  - Stop loss automático hardcoded
  - Max drawdown diario = X% → bot se detiene
  - Max posición = Y% del capital total
- [ ] Kill switch manual — parar todo con un comando

```
if (drawdown > MAX_DAILY_LOSS) → CLOSE_ALL → HALT → ALERT
```

---

#### M2 — Capa de IA / Razonamiento (Semana 5-6)
El bot no solo sigue reglas — razona y justifica cada decisión.

- [ ] Claude API integrado (Sonnet) para:
  - Generar narrativa del trade: "Entré porque..."
  - Evaluar contexto macro antes de ejecutar
  - Detectar condiciones de mercado anómalas
- [ ] Reasoning log: cada decisión tiene texto explicativo
- [ ] Este log es el script del contenido de Instagram

---

#### M3 — Observabilidad & Contenido (Semana 7-8)
El bot genera su propio contenido automáticamente.

- [ ] Dashboard en tiempo real (CLI-first)
- [ ] Auto-reporte diario: P&L, trades ejecutados, razonamientos
- [ ] Screenshot automático de cada trade (para reels)
- [ ] Notificación a WhatsApp/Telegram cuando ejecuta
- [ ] Balance diario generado sin intervención

---

### Stack técnico

| Componente | Tecnología | Razón |
|------------|-----------|-------|
| Runtime | Node.js | Ecosistema AIOX |
| IA / Razonamiento | Claude API (Sonnet) | Mejor reasoning |
| Storage | SQLite → PostgreSQL | Simple primero, escala después |
| Exchange | API nativa | Sin intermediarios |
| Observabilidad | Terminal dashboard | CLI First (Constitución) |

---

### Árbol de decisión de riesgo

```
¿Setup válido?
    NO → No entra. Log: "Esperando condición X"
    SÍ → ¿Drawdown diario < límite?
              NO → HALT. Alerta.
              SÍ → ¿Posición actual < max exposición?
                        NO → No entra
                        SÍ → EJECUTA → Log razón → Genera contenido
```
